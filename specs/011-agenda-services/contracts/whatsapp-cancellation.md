# Contract: WhatsApp "NÃO" Cancellation Flow

**Spec**: 011-agenda-services
**Depends on**: Spec 008 (WhatsApp Webhook), Spec 010 (Notifications/INotificationService).

This is **not** a public REST endpoint — it's a server-side flow triggered inside `WaWebhookProcessorJob.ProcessAsync` before the AI orchestrator is invoked.

---

## Trigger

Incoming WhatsApp text message that, after normalization (lowercase + diacritics removed), equals `"nao"`.

Detection happens in `ReminderResponseInterpreter.TryInterpretAsync(tenantSlug, conversationId, messageText)`:

```text
1. Normalize message: trim, lowercase, remove diacritics.
2. If normalized != "nao": return Outcome.NotApplicable.
3. SELECT appointments WHERE conversation_id = @conv
                          AND status = 'confirmed'
                          AND reminder_sent_at IS NOT NULL
                          AND reminder_sent_at >= now() - INTERVAL '26 hours'
                       ORDER BY start_at ASC
                       LIMIT 1
4. If no row: return Outcome.OutsideWindow (process as normal message).
5. Return Outcome.Cancelled(appointment).
```

---

## Cancellation execution

When `Outcome.Cancelled`, the webhook processor:

1. **Skips the AI orchestrator** for this message — the response is generated by the cancellation flow.
2. Executes `CancelAppointmentByClientCommand`:
   - Updates `appointment.status = 'cancelled'`, `cancelled_by = 'client'`, `cancelled_at = now()`, `cancellation_reason = 'Cliente cancelou via WhatsApp respondendo NÃO'`.
   - Appends `{slug}_appointment_events` with `action: "cancelled"`, `actor_type: "client"`, `metadata.channel: "whatsapp"`.
   - Loads `agenda_settings` for tenant.
3. Computes whether late cancel applies: `(appointment.start_at - now()) < late_cancel_window_hours`.
4. Renders response text:

```text
"Seu agendamento de {DD/MM/YYYY HH:mm} foi cancelado."

+ {cancellation_policy_text}     (if non-empty)

+ {late_cancel_text}             (if late cancel applies)
```

5. Enqueues the response via `OutgoingMessagePublisher` (Spec 008) with `sender_type = system`, `message_type = text` (free text within 24h session — no template).
6. Calls `INotificationService.NotifyAppointmentCancelledByClientAsync(ticketId, appointmentId, contactName, ct)` (Spec 010 — new method added by Spec 011) to alert the responsible attendant.
7. Emits `appointment.changed action: "cancelled"` WebSocket event (see `agenda-websocket-events.md`).

## Response message examples

### Inside late cancel window, no policy text:

```text
Seu agendamento de 10/06/2026 09:00 foi cancelado.

Cancelamentos com menos de 24h poderão ser cobrados.
```

### Inside late cancel window, with policy text:

```text
Seu agendamento de 10/06/2026 09:00 foi cancelado.

Política de cancelamento: agradecemos o aviso. Para reagendar, basta nos enviar uma mensagem.

Cancelamentos com menos de 24h poderão ser cobrados.
```

### Outside late cancel window:

```text
Seu agendamento de 15/06/2026 14:00 foi cancelado.

Política de cancelamento: agradecemos o aviso. Para reagendar, basta nos enviar uma mensagem.
```

## Error paths

| Condition | Behavior |
|---|---|
| Tenant has no `agenda_settings` row | Impossible — migration inserts default row. |
| WhatsApp channel inactive | Cancellation still executed; response not enqueued; log warning. |
| Multiple eligible appointments | Cancel only the earliest `start_at` (research §R11); response mentions specific date/time. |
| Race: appointment cancelled simultaneously by another path | Second cancel attempt sees `status != 'confirmed'` and returns `Outcome.NotApplicable`; message processed as normal. |
| Contact lacks phone in DB but message arrived | Cannot happen — webhook resolves contact by phone before processing. |

## INotificationService extension

```csharp
// Features/Notifications/INotificationService.cs (extended by Spec 011)

public interface INotificationService
{
    // ...existing Spec 010 methods...

    /// <summary>
    /// Spec 011 — alerts the responsible attendant when a client cancels an appointment
    /// via WhatsApp "NÃO" response. Fans out to ticket's attendant (if any) + supervisors
    /// of the department (mesma regra de SLA breach na Spec 010).
    /// </summary>
    Task NotifyAppointmentCancelledByClientAsync(
        Guid? ticketId,
        Guid appointmentId,
        string contactName,
        DateTimeOffset appointmentStartAt,
        CancellationToken ct);
}
```

The in-app notification appears as:

```text
Cliente cancelou agendamento via WhatsApp
{contactName} — {DD/MM HH:mm}
[ver agendamento]
```

Linked to `/agenda/{appointmentId}` in the CRM.

## Observability

Metrics:

- `reminder_response_no_total{tenant, outcome="cancelled"|"outside_window"|"no_match"}` — counter for every "NÃO" detected (regardless of outcome).
- `appointment_cancellations_total{tenant, by="client", channel="whatsapp"}` — counter for actual cancellations executed.

Logs (Serilog):

- INFO: "Cancellation via WhatsApp executed. Tenant={slug}, Appointment={id}, Contact={contactId}"
- WARN: "WhatsApp 'NÃO' received outside 26h window. Tenant={slug}, Conversation={convId}" — useful for tuning.
- ERROR: any exception bubbled up causes the message to be processed as normal text (graceful degradation).

## Implementation notes

- `Features/Agenda/Cancellation/ReminderResponseInterpreter.cs` — pure function (text-in, outcome-out). Easy to unit test.
- `Features/Agenda/Cancellation/CancelAppointmentByClientCommand.cs` — DI-injected; called by the webhook processor.
- `Features/WhatsApp/Webhook/WaWebhookProcessorJob.cs` — extended: after parsing the incoming text, calls `ReminderResponseInterpreter.TryInterpretAsync(...)`. If `Cancelled`, executes flow above and returns BEFORE calling `AgentOrchestrator`. Otherwise proceeds normally.
- Tests in `tests/Features/Agenda/Cancellation/` (see plan.md).
